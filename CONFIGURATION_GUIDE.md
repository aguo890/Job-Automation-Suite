# 🚀 New User Configuration Guide

Welcome to the Job Automation Suite! Before you can start automating your job search and generating tailored resumes, you need to configure your environment and inject your career history. 

Follow these four steps to get fully set up in less than 5 minutes.

---

## Step 1: Environment Setup & API Keys

The suite relies on AI models to power its automatic resume-tailoring algorithms. You need to configure a `.env` file at the root of the repository.

1. In your terminal or code editor, navigate to the root of the `Job-Automation-Suite` folder.
2. We have provided a template. Copy the `.env.example` to a functional `.env` file:
   ```bash
   cp .env.example .env
   ```
3. Open the newly created `.env` file. You must add **at least one** API key for the AI tailoring to work. You can use either OpenAI or DeepSeek:
   - `OPENAI_API_KEY=your_openai_key_here`
   - `DEEPSEEK_API_KEY=your_deepseek_key_here` (Recommended for cost-effective reasoning)

> [!NOTE]
> If you leave `USE_GITHUB_DATA=false` (the default), the app will use local storage for your jobs. If you are integrating with a deployment pipeline, set it to `true` and provide a `GITHUB_TOKEN`.

---

## Step 2: Preparing Your Master CV 📄

The automation suite generates beautiful PDF resumes using the [RenderCV](https://github.com/sinaatalay/rendercv) formatting engine. It requires a "Master" CV written in YAML.

1. Navigate to the `rendercv/` folder: `Job-Automation-Suite/rendercv/`
2. Create your Master CV YAML file:
   - Copy the provided template file: `cp master_cv.yaml.example my_cv.yaml`
   - We recommend renaming it to your name (e.g., `Aaron_Guo_CV.yaml`).
   - *Ensure the structure follows the RenderCV schema.*
3. When the AI generator kicks in, it reads this master file as the **Source of Truth**, selectively re-writing bullets and summaries based on the job posting, saving the new tailored version securely without destroying this master file.

> [!IMPORTANT]
> Your Master CV YAML file **must** be placed physically inside the `rendercv/` folder! Otherwise, the compiler will not detect it, and your PDF renditions will fail.

---

## Step 3: Defining Search Parameters 🎛️

Now it's time to tell the scraper what jobs you actually want. We've built an intuitive UI for this so you don't have to fiddle with terminal commands.

1. Boot up the Streamlit Dashboard (if you haven't already):
   ```bash
   make run
   ```
2. Navigate to **`http://localhost:8501`** in your browser.
3. Click on the **`⚙️ Configuration`** tab in the left-hand sidebar.
4. **Target Titles & Locations**: Enter the exact job titles (e.g., "Software Engineer", "Data Scientist") and locations (e.g., "New York", "Remote") you are targeting.
5. **Target Companies**: Navigate to the "Target Companies" sub-tab to view your active company roster.
6. **Blacklisted Words**: Use the Strict Filters to block titles with words like "Senior", "Principal", or "Staff" if you are looking for entry-level roles.

> [!TIP]
> **Advanced Users:** If you want to dump a list of 500 companies generated by an LLM, you can bypass the UI entirely. Just copy the templates in `job-scraping-app/config/` (e.g. `cp companies.yaml.example companies.yaml`) and edit the raw YAML files directly.

---

## Step 4: Your First Scrape 🚀

With your environment loaded, your CV injected, and your job constraints defined, you are ready to unleash the scraper.

1. In the Streamlit Dashboard, go to the **`🚀 Scraper`** tab.
2. Click **Start Scraping Run**.
3. Wait as the engine queries Ashby, Greenhouse, and Lever boards for your target companies.
4. Once completed, navigate to the **`📊 Dashboard`** tab. You'll see all your high-signal job matches ranked natively by your preset constraints.
5. Click on any job ID to enter the **Master CV Editor**, click **Auto-Tailor with AI**, and watch your PDF resume regenerate!

---

## Step 5: Maintenance & Upstream Syncing 🔄

To maintain a production-grade suite, you must keep your customized submodules synchronized with their original upstream repositories for bug patches and new features.

### The Local SRE Setup
Your environment is already configured with `upstream` remotes. You can check for updates using:
```bash
# For RenderCV
cd rendercv && git fetch upstream
# For Job Scraper
cd job-scraping-app && git fetch upstream
```

### Applying Updates

#### Option A: The "Surgical Strike" (Cherry-Pick)
Use this if you only want a specific fix or template without merging massive changes.
1. Find the commit hash from the upstream repository.
2. Run: `git cherry-pick <commit_hash>`

#### Option B: Full Synchronization (Merge)
Use this to bring your entire engine up to the latest version.
```bash
git merge upstream/main  # Or upstream/master for the scraper
```

> [!CAUTION]
> Always resolve merge conflicts in a visual editor like VS Code before committing! After merging inside a submodule, remember to run `git add <submodule_folder>` in the root directory to lock in the update.

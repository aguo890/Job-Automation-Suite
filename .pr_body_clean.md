

## Overview
This PR introduces a significant architectural upgrade to the AI-assisted CV Editor, transforming it from a "black-box" tailoring tool into a high-fidelity, transparent workflow engine. By decoupling the AI logic from the UI state and implementing a strategic density engine, we now guarantee professional, 1.5-page resumes optimized for specific Job Descriptions.

---

## 🚀 Key Features & Technical Depth

### 1. Intelligent Density Engine (The 1.5-Page Rule)
- **Subtractive Patching**: Refactored the core tailoring logic in `ai_tailor.py` to be **Length-Aware**.
- **Experience Strategy (Continuity)**: Retains 100% of employment history to avoid ATS chronological flags, but automatically strips bullets for low-signal roles to reclaim vertical space.
- **Project Strategy (Prioritization)**: Dynamically filters the projects section; irrelevant projects are omitted entirely to focus on high-correlation technical work.
- **Schema Safety**: Implemented robust handling for `highlights: []` arrays in the RenderCV/YAML pipeline.

### 2. Modernized CV Orchestration (`cv_bridge.py`)
- **Priority Discovery Engine**: Replaced brittle alphabetical globbing with a scoring engine that prioritizes personal master files (e.g., `Aaron_Guo_CV.yaml`) while blacklisting generic templates and temporary drafts.
- **AST-Based Theme Mutation**: Integrated `ruamel.yaml` to perform atomic updates to the `cv.theme` key in the Master CV, preserving 100% of human-written comments and formatting.
- **Native Overrides**: Leveraged RenderCV's dotted-path override syntax (`cv.sections.summary.0`) for manual targeting injections, bypassing the need for temporary YAML mutations in non-AI mode.
- **Atomic Save & Rotate**: Implemented timestamped `.bak` rotation logic during every Master CV save to prevent data loss.

### 3. High-Fidelity Status Monitoring & UI Sync
- **Live Lifecycle Logs**: Integrated Streamlit's `st.status` container with a custom `status_callback` pattern. Users now see real-time milestones: *DeepSeek R1 Handshake*, *JSON Parse*, *Hallucination Gate*, and *Finalization*.
- **Versioned Component Mounting**: Introduced `st.session_state["editor_version"]` to force a hard re-mount of the Ace Editor widget once AI operations finish, ensuring the buffer always matches the tailored state.
- **Exception Guardrails**: Comprehensive `try/except/finally` blocks ensure the UI unlocks and reports status even in the event of API timeouts or network transients.

### 4. AI Resilience & Sanitization Layer (`core/`)
- **Global Sanitizer**: Extracted "Healer" logic into a top-level `core/llm_sanitizer.py` utility.
- **Format-Agnostic Healing**: Automatically strips DeepSeek reasoning blocks (`<think>`) and citation artifacts (`[cite_start]`) from both JSON (Tailoring) and YAML (Onboarding) payloads.
- **Smart Whitespace Management**: Implemented intelligent indent-safe removal of AI artifacts to prevent parsing failures without manual user correction.
- **Eradicated Brittle Logic**: Replaced all instances of legacy string splits and ad-hoc regex with the centralized resilience layer.
